# 面试算法题背诵手册 - 优先级队列问题

## 题目回顾

根据队列中的元素的优先级实现插队，优先级高的先出队列，如果优先级相同，入队时间早的先出队，有一个超时时间，如果队列中存在某个元素到达超时时间还未出队，则该队列转变成为普通先入先出队列，直到队列中不存在任何元素后，队列再次变为优先队列。

**难度：困难**

---

## 一、核心思路（背诵版）

### 整体理解
这个题目的本质是：**一个会变身的队列**
- 正常情况：是优先队列（VIP插队）
- 超时后：变身普通FIFO队列（人人平等）
- 清空后：恢复优先队列（恢复VIP插队）

### 关键要点（必背！！！）

1. **数据结构设计**
   - 每个元素要包装起来，包含：
     - `value`：实际数据
     - `priority`：优先级 ↑
     - `enqueueTime`：入队时间戳 ↓
     - `expiredTime`：过期时间

2. **状态管理**
   - `isFIFO`：标记当前是FIFO模式还是优先级模式
   - 只要有一个元素超时，整个队列切换为FIFO模式
   - 队列清空后，重置为优先级模式

3. **出队逻辑（核心中的核心）**
   ```
   if (isFIFO) {
       直接出队第一个元素
   } else {
       找没有过期的元素中优先级最高的
       如果有多个相同优先级，找入队时间最早的
       如果所有元素都过期了，切换到FIFO模式
   }
   ```

4. **检查超时的时机**
   - 每次出队时检查当前时间
   - 只要有一个元素过期，立即切换FIFO模式

## 二、暴力求解法（最简单但正确的实现）

### 代码结构（需要背框架）

```java
// 包装类 - 必须记住
class Element {
    Object value;
    int priority;
    long enqueueTime;
    long expiredTime;

    // 比较规则：优先级高的优先，相同优先级入队早的优先
    int compareTo(Element other) {
        if (this.priority != other.priority) {
            return other.priority - this.priority; // 降序
        }
        return (int) (this.enqueueTime - other.enqueueTime); // 升序
    }
}

// 主队列类
class FlexibleQueue {
    private LinkedList<Element> list = new LinkedList<>();
    private boolean isFIFO = false;
    private long currentTime = 0;

    // 入队
    void enqueue(Object value, int priority, long timeout) {
        Element e = new Element();
        e.value = value;
        e.priority = priority;
        e.enqueueTime = System.currentTimeMillis();
        e.expiredTime = e.enqueueTime + timeout;
        list.add(e);
    }

    // 出队 - 最重要！！！
    Object dequeue() {
        if (list.isEmpty()) return null;

        // 检查是否切换到FIFO模式
        checkAndSwitchToFIFO();

        if (isFIFO) {
            return list.removeFirst().value;
        } else {
            // 暴力求解：遍历所有元素找到最优解
            int bestIndex = -1;
            Element best = null;

            for (int i = 0; i < list.size(); i++) {
                Element e = list.get(i);
                // 跳过过期元素（理论上不应该有，除非检查间隔太长）
                if (e.expiredTime <= currentTime) continue;

                if (best == null || e.compareTo(best) > 0) {
                    best = e;
                    bestIndex = i;
                }
            }

            if (bestIndex == -1) {
                // 所有元素都过期，切换FIFO模式
                isFIFO = true;
                return list.removeFirst().value;
            }

            return list.remove(bestIndex).value;
        }
    }

    // 检查是否切换FIFO模式
    private void checkAndSwitchToFIFO() {
        currentTime = System.currentTimeMillis();
        for (Element e : list) {
            if (e.expiredTime <= currentTime) {
                isFIFO = true;
                return;
            }
        }
    }

    // 清空后重置
    void resetIfEmpty() {
        if (list.isEmpty()) {
            isFIFO = false;
        }
    }
}
```

### 为什么这是暴力解法？

1. **遍历查找**：每次出队都遍历整个列表找最优元素 → O(n)
2. **线性检查**：每次都要检查所有元素是否过期 → O(n)
3. **简单容器**：直接使用LinkedList，不做复杂索引

**优点**：思路简单，容易记忆，杜绝bug
**缺点**：性能差，大数据量时慢，但面试够用！

## 三、面试背诵要点（按顺序背）

### 步骤1：设计数据结构（说给面试官听）
"这个题目需要维护一个会变身的队列，首先我会设计一个包装类，包含原始值、优先级、入队时间和过期时间四个字段..."

### 步骤2：描述状态机（核心！）
"队列有两种状态：
- 正常状态：按优先级+入队时间排序出队
- FIFO状态：按入队顺序出队

状态转换触发条件：只要有一个元素超时 → 立即切换到FIFO
状态重置条件：队列清空 → 恢复优先级模式"

### 步骤3：出队算法（慢慢说，边想边说）
```
if (isFIFO) {
    取出第一个元素
} else {
    遍历所有元素：
        跳过过期元素
        找优先级最高的
        优先级相同找入队最早的
    如果没找到：切换FIFO，取第一个
}
```

### 步骤4：时间复杂度（面试常问）
- 入队：O(1)
- 出队：O(n) —— 因为是暴力遍历
- 空间：O(n)

**如果被问优化**：
"可以使用优先队列+延迟删除优化到O(log n)，但实现复杂容易出错。目前的暴力解法在代码清晰度和正确性之间取得了平衡。"

## 四、可能的追问和准备答案

### Q1：为什么用LinkedList？
A：因为需要删除任意位置的元素。ArrayList删除是O(n)，LinkedList理论上也是O(n)因为要遍历，但删除本身比较快。

### Q2：如何优化性能？
A：(反向思考)
```
方案1：维护一个优先队列 + 延迟删除
- 入队直接进优先队列 O(log n)
- 出队时检查队首是否过期，如果是就丢弃 O(log n)
- 但需要处理队首过期导致切换FIFO的情况

方案2：两个队列
- 一个优先队列（未过期）
- 一个FIFO队列（过期后整体转移）
- 状态更清晰
```

### Q3：并发场景怎么处理？
A：（高开低走法）
"首先我会用synchronized或ReentrantLock保证线程安全...（然后简单描述一下加锁范围）"

### Q4：如果超时时间很长怎么办？
A：（诚实回答）
"如果超时时间很长，队列长时间在优先级模式下运行，我的暴力解法效率会低一些。但实际项目中，超时通常是秒级或分钟级，不会有明显性能问题。"

## 五、手写版（面试时快速写出）

```java
public class DynamicPriorityQueue {
    static class Node {
        String value;
        int priority;
        long inTime;
        long expTime;
    }

    private LinkedList<Node> data = new LinkedList<>();
    private boolean isFifo = false;

    public void add(String v, int p, long t) {  // 入队
        Node n = new Node();
        n.value = v;
        n.priority = p;
        n.inTime = System.currentTimeMillis();
        n.expTime = n.inTime + t;
        data.add(n);
    }

    public String poll() {  // 出队
        if (data.isEmpty()) return null;

        if (!checkTimeout()) {  // 检查是否该切换FIFO
            isFifo = true;
        }

        if (isFifo) {
            return data.removeFirst().value;
        }

        // 遍历找最优
        Node best = null;
        int idx = 0, bestIdx = -1;
        for (Node n : data) {
            if (n.priority > best.priority ||
                (n.priority == best.priority && n.inTime < best.inTime)) {
                best = n;
                bestIdx = idx;
            }
            idx++;
        }

        return data.remove(bestIdx).value;
    }

    private boolean checkTimeout() {
        long now = System.currentTimeMillis();
        for (Node n : data) {
            if (now >= n.expTime) return false;
        }
        return true;
    }
}
```

## 六、背诵口诀

**核心4句：**
1. 元素包装四件套（值、优先级、入队时间、过期时间）
2. 一状态二模式（isFIFO标志位）
3. 出队先看状态（是FIFO就取头，不是就遍历）
4. 超时切FIFO，空队列重置

**时间复杂度**：
入O(1)、出O(n)、空O(1)

**追问怎么答**：优先保证正确，然后说优化思路
